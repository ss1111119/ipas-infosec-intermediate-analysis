題號：4｜iPAS 資訊安全防護實務

檔名：ipas_web_app_defense_q04.html
為了防範常見的網站應用程式攻擊，如 SQL 注入（SQL Injection）與跨網站指令碼（XSS），開發團隊在設計安全架構時，應實施下列哪些防護與應變技術最為適切？（複選）
【題目】

為了防範常見的網站應用程式攻擊，如 SQL 注入（SQL Injection）與跨網站指令碼（XSS），開發團隊在設計安全架構時，應實施下列哪些防護與應變技術最為適切？（複選）

【第一步：找關鍵字】

請用螢光筆標示出考題重點：

SQL Injection、 XSS、 複選、 最為適切、 伺服器端、 防護、 參數化查詢、 CSP、 輸出編碼、 HTTP/HTTPS


核心提示：本題考的是「多層防護（Defense in Depth）」觀念，針對不同的攻擊手法（SQLi, XSS）需要對應不同的防禦層（資料庫層、應用層、瀏覽器層）。

【第二步：白話翻譯】
高中生日常語言改寫：

「如果要防止網站被壞人塞惡意指令（弄壞資料庫）或偷資料（騙使用者），開發網站時有哪些安全措施是一定要做的？」


名詞解釋：
  • SQL 注入 (SQLi)： 壞人把惡意的資料庫指令塞進輸入框，如果你沒檢查，資料庫就會乖乖執行，導致資料被刪光或被看光。
  • XSS (跨網站指令碼)： 壞人把惡意 JavaScript 程式碼塞進網頁留言板，當別人瀏覽該頁面時，瀏覽器就會執行這段程式，導致帳號被盜。
  • CSP (內容安全策略)： 告訴瀏覽器一張「白名單」，規定只能載入哪些地方的程式碼，防止執行來路不明的惡意腳本。
【第三步：解題思路】

這題在考什麼？ 常見網站攻擊需用「多層正確對策」防護，不可因效能或方便而犧牲安全。


解題邏輯：
  1. 對照攻擊類型： 題目提到 SQL Injection 和 XSS。
  2. 判斷選項有效性：
    • SQLi 的剋星是「參數化查詢」。
    • XSS 的防護主力是「輸出編碼」，輔助防護是「CSP」。
  3. 剔除毒藥選項： 任何建議「不使用加密（HTTPS）」的選項，在現代資安觀念中絕對是錯的。
【第四步：選項分析】
  • (A) 正確： 採用「參數化查詢」可以確保資料庫將使用者的輸入視為純文字，而非指令。這是防範 SQL Injection 最有效且標準的方法。
  • (B) 正確： CSP 是瀏覽器端的防線，能限制外部腳本來源，即使網站有漏洞，也能大幅降低 XSS 攻擊成功的機率。
  • (C) 正確： 「輸出編碼」將特殊字元（如 <, >, "）轉換為 HTML 實體（如 &lt;），讓瀏覽器只顯示文字而不執行程式碼，是防範 XSS 的根本之道。
  • (D) 錯誤： HTTP 明文傳輸極度危險，容易遭中間人竊聽或竄改。現代硬體強大，HTTPS 的效能損耗幾乎可忽略不計，絕不能為了微小的效能而犧牲加密。
【觀念補充與延伸】
核心觀念：多層防護 (Defense in Depth)

不要只依賴單一防線，必須在不同層級設置關卡。


SQLi / XSS 對策對照卡

SQL Injection：

  • 主防：參數化查詢 (Prepared Statements)
  • 輔防：輸入驗證、最小權限原則

XSS：

  • 主防：輸出編碼 (Output Encoding)
  • 輔防：CSP、HttpOnly Cookie

防護層級分類

  • 資料庫層： 參數化查詢
  • 應用層： 輸出編碼、輸入驗證
  • 傳輸層： HTTPS (TLS)
  • 瀏覽器層： CSP
【記憶口訣】

「SQL 用參數，XSS 要編碼；
CSP 再補強，HTTP 別亂上。」

「三層一起來，網站才安全」：伺服器層(A)、瀏覽器層(B)、應用邏輯層(C)。

【常見陷阱】
  • 陷阱一： 以為只做「輸入驗證（過濾）」就夠了，忽略了駭客可能繞過過濾規則，必須搭配「輸出編碼」。
  • 陷阱二： 誤信「HTTPS 會拖慢網站」的過時觀念，而在題目中選擇放棄 HTTPS。
  • 陷阱三： 複選題只選了一個最熟悉的選項（例如只選 A），忘記資安強調全面防護。
  • 避坑指南： 看到選項寫「僅在 HTTP」或「避免加密」，幾乎 100% 是錯誤選項。
【正確答案】

A、B、C

需同時選取三者才算完整得分。

【自我檢測】

請嘗試回答以下問題（不提供標準答案，請反思）：

  1. 為什麼只做輸入驗證（Input Validation）仍可能發生 XSS？（提示：資料庫可能已經有髒資料）
  2. CSP 能「完全防止」XSS 嗎？為什麼？（提示：如果網站自己內嵌了不安全的 inline script 怎麼辦？）
  3. 若網站未使用 HTTPS，除了資料被竊聽，還可能衍生哪些風險？（提示：中間人攻擊、Session Hijacking）
