第 11 題

主題：存取控制方法：MAC / ACL / RBAC

看到「Randomly」要提高警覺（陷阱名詞）
題目：「關於『資料』存取的控制方法，『不』包括下列何者？」
點此展開完整教學解析
【題目】

關於「資料」存取的控制方法，「不」包括下列何者？
(A) 強制存取控制（Mandatory Access Control，MAC）
(B) 存取控制清單（Access Control List，ACL）
(C) 隨機存取控制（Randomly Access Control，RAC）
(D) 角色基準存取控制（Role-based Access Control，RBAC）

【第一步：找關鍵字】

用粗體標出並解釋線索：

  • 資料存取的控制方法：題目在問「有哪些正式的方法/模型」。
  • 不包括：反向題，找「不是正規名詞」。
  • MAC / ACL / RBAC：這三個是資安課本常見標準詞。
  • Randomly：隨機。在安全控管上通常不是正規模型名稱，常用來設陷阱。
【第二步：白話翻譯】

用高中生日常口語說：

「哪些方法可以決定『誰能看/改這份資料』？下面哪個聽起來像名詞，但其實不是正規的存取控制方法？」

【第三步：解題思路】

這題在考什麼觀念？
→ 辨識常見存取控制模型與機制名稱，排除不存在的假名詞。

解題步驟：

  1. 先把你熟悉、課本常見的存取控制模型圈起來（MAC、RBAC）。
  2. 再看控制實作機制（ACL）。
  3. 剩下那個若是「不像正式術語」通常就是答案（RAC）。

原理與規則（名詞第一次出現要括號白話解釋）：

  • 強制存取控制 MAC（由系統強制規則，使用者不能自己改權限）
  • 角色基準存取控制 RBAC（把權限發給「角色」，人再被指派到角色）
  • 存取控制清單 ACL（一份名單，列出誰對某資源有什麼權限）
  • 隨機存取控制 RAC（題目中的 Randomly Access Control 不是常見標準模型；真正常見的是 DAC/MAC/RBAC/ABAC 等）
【第四步：選項分析】
  • (A) MAC：為什麼是正規方法？（系統強制規則，常見模型）
  • (B) ACL：為什麼算控制方法？（實作權限設定與檢查的常見機制）
  • (C) RAC：為什麼不包括？（不是標準存取控制模型/機制名稱，屬干擾選項）
  • (D) RBAC：為什麼是正規方法？（以角色管理權限，企業常用）
【觀念補充與延伸】 (★重要)

核心觀念定位：
- 屬於「存取控制模型（Access Control Models）」的高頻必考題型，常用來考名詞辨識與概念對比。

法條或定義（學術定義＋白話）：
- 存取控制（Access Control：決定誰能對資料/系統做什麼動作）
- MAC/RBAC/ACL 的簡要定義如上所述。

觸類旁通（舉一反三）：

  • 對比概念：DAC（自主存取控制：資源擁有者可授權） vs MAC（系統強制）
  • 變化題型：
    1. 「下列何者屬於存取控制模型？」（模型 vs 機制）
    2. 「RBAC 的優點是什麼？適合什麼情境？」（角色多、權限管理複雜的組織）
【記憶口訣】
  • 「三真一假：MAC、ACL、RBAC 是真的；RAC 多半是假的。」
  • 「Randomly 聽起來就不安全：不會拿來當正式控管模型名。」
【常見陷阱】

最容易搞混：
- 以為每個縮寫都是真的，看到 RAC 就直覺覺得是新名詞。

心理盲點：
- 英文縮寫＋中文翻譯看起來很像正式術語，容易被騙。

避坑指南：
- 考試快速檢查：
1) 你背過的四大模型通常是 DAC / MAC / RBAC / ABAC
2) 不在這個範圍內的，多半是干擾（除非題目明確教新名詞）

【正確答案】

答案：(C)

一句話理由：RAC（Randomly Access Control）不是常見或標準的資料存取控制模型/機制，屬干擾選項。

【自我檢測】

設計 3 題延伸小問題（點擊題目查看答案）：

1) RBAC 為什麼適合企業？
答：用角色集中管理權限，員工異動時只要換角色即可
2) MAC 和 DAC 最大差異是什麼？
答：MAC 由系統強制；DAC 可由資源擁有者自行授權
3) ACL 比較像「模型」還是「實作機制」？
答：比較像實作機制/權限清單的表達方式

【一頁速記卡】存取控制關鍵字

MAC 系統強制規則（使用者不能隨便改）
RBAC 角色管理權限（人→角色→權限）
ACL 針對資源列權限清單（誰能做什麼）
常見陷阱 RAC（Randomly）不是正規模型
