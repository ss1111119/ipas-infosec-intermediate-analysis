iPAS 資訊安全（規劃/防護）考題解析

題目 10（單選題）

「關於身分識別與存取管理（Identity and Access Management，IAM）的敘述，下列何者正確？」

  • (A) 管理網路防火牆的設定
  • (B) 控制使用者對系統或資源的存取權限
  • (C) 監控網路流量與入侵偵測
  • (D) 保護敏感資料的傳輸
第一步：找關鍵字

請將目光集中在題目的這些字眼上：

「關於 身分識別 與 存取管理（Identity and Access Management，IAM）的敘述...」

為什麼是線索？

  • IAM 是資訊安全領域中一個特定的管理框架，題目明確在問這個框架的「職責範圍」。
  • 這不是在問所有的資安工具，而是專指針對「人（身分）」與「權限（存取）」的管理。
第二步：白話翻譯

如果用高中生的日常口語來說，這題其實是在問：

「IAM（Identity and Access Management）這個東西到底是負責管什麼的？是不是就是在管『誰可以登入系統』以及『登入後可以做什麼事』？」
第三步：解題思路

這題在考什麼觀念？

這題測驗你是否理解 IAM（身分識別與存取管理：管理「誰」能登入、登入後「能做什麼」）的核心職能。它的核心是處理「人」與「資源」之間的關係，而不是處理網路硬體設備或傳輸加密。

解題步驟

  1. 看到 IAM，就要想到兩件事：Authentication（驗證你是誰） 和 Authorization（授權決定你能做什麼）。
  2. 檢查四個選項，哪一個選項是在講「使用者」和「權限」？
  3. 排除那些屬於網路層（防火牆）、監控層（IDS）或傳輸層（加密）的選項。

原理與規則（生活比喻）

想像一間門禁森嚴的大型公司：

  • IAM 就像是「警衛室發放識別證的系統」。警衛確認你是員工（身分識別），然後在你的卡片裡設定權限（存取管理），讓你可以刷卡進入辦公室，但不能進入金庫。
  • 如果沒有 IAM，就像任何人都可以隨意進出所有房間，非常危險。
第四步：選項分析
  • (A) 管理網路防火牆的設定 [錯誤]
    理由：這是 防火牆（管網路封包要不要進出） 的工作。防火牆主要看 IP 位址和連接埠，通常不直接管使用者的細部權限。
  • (B) 控制使用者對系統或資源的存取權限 [正確]
    理由：這正是 IAM 的定義。它包含「認證使用者身分」以及「授予適當的 存取權限（允許或禁止你讀/寫/刪/執行）」。
  • (C) 監控網路流量與入侵偵測 [錯誤]
    理由：這是 IDS/入侵偵測（看有沒有可疑攻擊行為） 或 SOC（資安維運中心）的工作範圍，重點在於發現攻擊，而非管理帳號權限。
  • (D) 保護敏感資料的傳輸 [錯誤]
    理由：這是 加密傳輸（把資料在路上變成看不懂，防偷看） 的技術，例如 VPN 或 TLS/SSL 協定。雖然 IAM 系統登入時會用到加密，但「保護傳輸」本身不是 IAM 的核心定義。
觀念補充與延伸

核心觀念定位

IAM 是企業資安的第一道防線（身分邊界）。即使網路防火牆被突破，如果 IAM 做得好，駭客拿到帳號也未必能存取核心資料。

資安職能對照表

為了避免混淆，請參考下表區分不同資安工具的職責：

工具/技術 核心職責 關鍵字
IAM 管理 人 與 權限 帳號、密碼、登入、授權
Firewall (防火牆) 管理 封包 進出 IP、Port、阻擋連線
IDS (入侵偵測) 偵測 可疑行為 監控、警報、異常流量
Encryption (加密) 保護 內容 不外洩 亂碼、金鑰、傳輸安全

觸類旁通：Authentication vs Authorization

IAM 系統通常包含兩個關鍵流程（常考比較）：

  1. 驗證 (Authentication)：
    定義：證明你真的是你。
    例子：輸入密碼、指紋辨識、收到手機簡訊驗證碼 (OTP)。
  2. 授權 (Authorization)：
    定義：決定你能用哪些功能或資料。
    例子：你是「工讀生」，所以你只能「讀取」文件，不能「刪除」文件。
記憶口訣

為了快速記住 IAM 的功能，請記住這四個字：

IAM = 「認人 ＋ 給權」
  • 認人：確認身分（身分識別）。
  • 給權：控制你能做什麼（存取管理）。
常見陷阱

為什麼容易搞混？

初學者常誤以為「只要是保護系統不被亂用的就是 IAM」。其實資安是分層防禦的：

  • 如果你是在管「網路線路」誰能通，那是防火牆。
  • 如果你是在管「使用者帳號」誰能登入，那才是 IAM。

避坑檢查步驟

看到題目選項時，問自己一個問題：「這個功能是在管 IP 位址（機器），還是在管帳號（人）？」如果是管 IP，通常不是 IAM。

正確答案

正確答案：(B)

一句話理由：因為 IAM 的核心定義就是管理使用者的身分識別以及控制其對資源的存取權限。

自我檢測

請嘗試回答以下三題延伸題，點擊按鈕查看答案。

Q1. 小明輸入帳號密碼登入公司的網站，這個動作屬於 IAM 中的哪一個階段？（Authentication 還是 Authorization？）

答案：Authentication（驗證/認證）。因為這是證明他是小明的過程。

Q2. 下列哪一項最不屬於 IAM 的功能範疇？
A. 強制使用者每 90 天更換密碼
B. 設定會計部門只能讀取財務報表，不能修改程式碼
C. 阻擋來自特定惡意 IP 的所有網路連線

答案：C。阻擋 IP 是防火牆的功能，不是 IAM（人與權限）的主要功能。

Q3. 在 IAM 的原則中，為了安全起見，我們應該給予使用者「最大權限」還是「最小權限」？

答案：最小權限（Least Privilege）。使用者只應獲得完成工作所需的最小權限，以降低風險。
