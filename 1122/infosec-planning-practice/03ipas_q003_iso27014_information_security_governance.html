iPAS 資訊安全工程師（初級）考題詳解

主題：資訊安全治理與標準框架

1) 【題目】題號：3

題型：單選題

題目：
「近年來，民間企業數位網通服務應用加驟，讓企業的資安單位從原本的支援角色轉變為重要的策略單位，而資訊安全『管理』更拉高至『治理』位階，下列哪一項可以作為企業治理資訊安全的框架及指引？」

  • (A) ISO/IEC 27004
  • (B) ISO/IEC 27011
  • (C) ISO/IEC 27014
  • (D) ISO/IEC 27017
2) 【第一步：找關鍵字】

請注意題目中的以下字詞：

「...而資訊安全『管理』更拉高至『治理』位階，下列哪一項可以作為企業治理資訊安全的框架及指引？」

為什麼這是線索？
  • 治理 (Governance)： 這代表層級已經不是單純的技術執行或部門管理，而是上升到了「董事會」或「最高管理階層」的戰略層面。
  • 框架及指引： 題目在問針對這個高層級（治理層），ISO 標準家族中哪一份文件是專門用來指導這些高層長官如何監督與評價資安的。
3) 【第二步：白話翻譯】

用最通俗的話來說，這題在問：

「現在資安很重要，不只是下面的人在做，上面的大老闆也要管。請問如果要教『董事會/老闆』怎麼制定方向跟監督資安，應該要參考哪一本說明書？」

4) 【第三步：解題思路】

這題在考什麼觀念？

這題的核心考點是區分「資訊安全治理 (Information Security Governance)」與「資訊安全管理 (Information Security Management)」的差異，並認識對應的 ISO 標準。

解題步驟：

  1. 辨識層級：題目強調「從管理拉高至治理」。
  2. 對應觀念：治理 (Governance) 是屬於組織最頂層（董事會）的責任，負責評估 (Evaluate)、指導 (Direct) 與監督 (Monitor)。
  3. 篩選標準：在 ISO 27000 系列中，尋找專門講述「Governance (治理)」的標準編號。

原理與規則（生活比喻）：

  • 治理 (Governance)： 就像是一艘船的「船長或船東」，決定船要開往哪裡（方向）、遇到風浪的策略是什麼（策略），並確保船員有在工作（監督）。
  • 管理 (Management)： 就像是「大副或輪機長」，根據船長的命令，制定排班表（流程）、操作機器（技術）、維護設備（執行），確保船隻照著方向前進。
5) 【第四步：選項分析】
  • (A) ISO/IEC 27004： [錯誤]
    這是關於 資訊安全管理量測 (Information Security Management — Monitoring, measurement, analysis and evaluation)。
    用途：教你如何設定 KPI、如何量化資安成效（例如：今年發生幾次資安事件、修補率多少）。它是用來「打分數」的工具，屬於管理層面的量測，非治理框架。

  • (B) ISO/IEC 27011： [錯誤]
    這是 ISO/IEC 27011（電信產業的資安管理指引）。
    用途：針對「電信業者」特製的資安實作指引。如果題目沒提到電信業，通常不選這個。

  • (C) ISO/IEC 27014： [正確]
    這是 ISO/IEC 27014（資訊安全治理的框架與指引）。
    用途：標題直接包含 "Governance of information security"。它定義了治理主體（董事會）如何評估、指導、監督資安活動，確保資安目標與企業商業目標一致。完全符合題目描述。

  • (D) ISO/IEC 27017： [錯誤]
    這是 ISO/IEC 27017（雲端服務的資安控制指引）。
    用途：針對「雲端服務提供者與使用者」的資安控制措施。題目未提及雲端環境，故不選。
6) 【觀念補充與延伸】

核心觀念：治理 vs 管理 對照表

維度 資訊安全治理 (Governance) 資訊安全管理 (Management)
負責人 董事會、高階管理層 (Top Management) 資安長 (CISO)、資安經理、IT 部門
核心動作 評估 (Evaluate)、指導 (Direct)、監督 (Monitor) — 簡稱 EDM 規劃 (Plan)、執行 (Do)、查核 (Check)、行動 (Act) — 簡稱 PDCA
關注焦點 策略一致性、價值交付、責任歸屬 減緩風險、落實控制措施、日常營運

ISO 27000 系列家族速查

編號 關鍵字/用途
ISO 27001 驗證標準（唯一可被驗證的標準），ISMS 的要求事項 (Requirements)
ISO 27002 實作指引 (Code of practice)，列出控制措施的詳細作法
ISO 27004 量測與績效 (Monitoring & Measurement)，講 KPI 和指標
ISO 27005 風險管理 (Risk Management)，教你怎麼做風險評鑑
ISO 27014 資安治理 (Governance)，給高層看的
ISO 27017 雲端安全 (Cloud Services)
ISO 27018 雲端個資保護 (PII in Cloud)
7) 【記憶口訣】

如何一秒記住 27014 = 治理 (Governance)？

口訣：「14」諧音「意思」

老闆的「意思」（14）就是聖旨（治理）

解析：治理是由上而下的，老闆（董事會）的意志（意思）決定了公司的方向，所以 27014 是關於治理。

8) 【常見陷阱】

學生容易混淆點：

  1. 誤以為 27001 就是治理：
    修正：27001 是建置整個系統 (ISMS) 的標準，雖然包含管理審查，但專門談「治理架構」的是 27014。
  2. 數字混淆：
    避坑方法：
    • 看到「雲端」找 17 (一起上雲端) -> 27017
    • 看到「量測/績效」找 4 (Measure 有 4 個音節 / 或者是 Performance 指標) -> 27004
    • 看到「治理/董事會」找 14 (老闆的意思) -> 27014
9) 【正確答案】

正確答案：(C) ISO/IEC 27014

一句話理由：因為 ISO/IEC 27014 是 ISO 家族中唯一專門針對「資訊安全治理 (Governance)」所制定的框架與指引標準。

10) 【自我檢測】

請嘗試回答下列延伸題目，點擊按鈕查看答案。

Q1. 公司最近導入了 AWS 與 Azure 服務，資安官希望找一份針對「雲端服務」的資安控制措施指引來參考，請問該查閱哪份標準？

答案：ISO/IEC 27017

解析：27017 是專門針對雲端服務提供者與客戶的資安控制指引。

Q2. 董事會要求資安長定期回報資安推動的成效，並要求將成效「量化」（例如數據化指標）。資安長應該參考哪份標準來設計這些績效指標？

答案：ISO/IEC 27004

解析：27004 是關於監控、量測、分析與評估 (Monitoring, measurement, analysis and evaluation) 的標準。

Q3. 某電信公司想要強化其特定產業的資安管理，除了 ISO 27001 之外，最應該參考哪一份針對電信業的實務指引？

答案：ISO/IEC 27011

解析：27011 是基於 ISO 27002 並針對電信組織 (Telecommunications organizations) 增修的資安管理指引。
