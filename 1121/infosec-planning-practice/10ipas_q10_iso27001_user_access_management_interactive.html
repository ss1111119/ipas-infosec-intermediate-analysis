iPAS 資訊安全衝刺班

ISO 27001 實務解析：使用者存取管理

一分鐘速記小卡

1. 核心原則：需知需用 (Need-to-know)。

2. 帳號生命週期：建立 -> 變更 -> 停用/移除。

3. 離職處理：立即停用帳號是第一要務。

4. 關鍵字連結：看到「離職仍可登入」 -> 直覺選「使用者存取管理」。

5. 資產處置：是指設備報廢或資料銷毀，勿混淆。

【題目】模擬試題 Q10

10. 一個通過 ISO 27001:2013 驗證的組織，在內部稽核活動中，發現已離職員工的帳號仍沒有停用，且進一步追查存取紀錄時，發現其在離職後，仍有登入存取部門共用資料夾的行為。請問對於該組織上述的狀況，下列何項敘述最為合適？

【第一步：找關鍵字】

請注意題目中標示的線索：

  • ISO 27001:2013 驗證：代表該公司「理論上」應該有完善的制度，但實際上可能未落實。
  • 內部稽核：這是用來「抓包」制度是否真的有在運作的檢查活動。
  • 已離職員工：身分關鍵，已經不具備存取權限的人。
  • 帳號仍沒有停用：問題核心，違反了帳號管理的原則。
  • 離職後仍有登入行為：造成的後果，發生了未授權存取。
  • 存取部門共用資料夾：雖然提到資料夾，但根源是因為「帳號」沒關掉。
【第二步：白話翻譯】

用高中生日常口語來說就是：

「人都已經離職走人了，結果他的帳號（鑰匙）居然還能用，甚至還能大搖大擺地開門進去部門資料夾看東西。這代表這家公司管帳號的人在睡覺，管理流程出大包了。」
【第三步：解題思路】

這題在考什麼觀念？

一句話：ISO 27001 中「使用者存取管理」是否落實。


解題步驟：

  1. 先確認事件重點：離職 ≠ 帳號停用（這是異常狀態）。
  2. 再對照 ISO 27001 的控制面向：屬於帳號生命週期管理。
  3. 排除與帳號無直接關聯的控制措施（如設備報廢、門禁刷卡）。

原理與規則（為什麼這樣判斷）：

  • 帳號是「人」存取系統的鑰匙，人一離職，鑰匙就要收回。
  • 若未即時停用，代表存取管理流程有漏洞，不符合規範。
【第四步：選項分析】
請先完成【題目】作答，正確後即可解鎖完整解析。
  • (A) 為什麼對？
    因為未依人員異動（離職）即時停用帳號，這正是「使用者存取管理」章節中關於「使用者存取權限移除與調整」的規範失效。
  • (B) 為什麼錯？
    「資產處置」通常是指電腦設備報廢、硬碟銷毀、紙本文件碎紙等。雖然帳號也是資產，但在 ISO 條文中，帳號權限問題專屬於存取控制領域。
  • (C) 為什麼錯？
    「實體及環境安全」管的是門禁刷卡、機房上鎖、滅火器、攝影機等物理設備。題目強調的是「系統帳號登入」，屬於邏輯安全，非實體安全。
  • (D) 為什麼錯？
    事件已經顯示出嚴重漏洞（離職還能登入），這證明了相關政策「沒有」被落實，而不是已落實。
【觀念補充與延伸】

核心觀念定位：

  • 屬於「存取控制與帳號生命週期管理」章節。
  • 這是 ISO 27001 驗證與 iPAS 考試的必考題型。

標準重點或定義：

  • 使用者存取管理 (User Access Management)： 確保只有經過授權的使用者才能存取系統。包含帳號的建立、權限變更、密碼管理、以及最重要的「離職/異動時的停用」。
  • 帳號生命週期： 到職(申請) → 在職(使用/變更) → 離職(停用/刪除)。

觸類旁通：

  • 對比概念： 人員離職 vs 系統帳號停用（兩者必須同步）。
  • 變化題型： 如果題目改成「部門異動後，仍擁有原部門權限」，這叫做「權限未移除/權限蔓延」，答案一樣選「存取管理」相關選項。
【記憶口訣】

「人走帳號留，一定管不周」

(人離職了帳號還留著，就是管理不周全)

【常見陷阱】

避坑指南：看到「帳號還能登入」，第一時間就想到「使用者存取管理」。

容易混淆點：

  1. 看到「共用資料夾」誤以為是「資產處置」問題。（資料夾是標的，帳號才是手段）
  2. 看到「通過 ISO 27001 驗證」就直覺以為制度一定有落實。（題目正是要告訴你，驗證過了也可能有疏漏，所以才需要內部稽核）。
  3. 混淆「帳號管理」與「實體安全」。（記得：軟體/系統歸邏輯存取，門禁/硬體歸實體安全）。
【正確答案】

(A)

一句話理由：離職帳號未停用，代表使用者存取管理流程未落實。

【自我檢測】

點擊問題查看答案：

Q1) 若是「帳號已停用，但門禁卡仍可進公司」，應歸類哪一類問題？

A: 實體及環境安全 (Physical Security)。因為門禁卡控制的是實體場域的進出。

Q2) 若是「臨時帳號到期後仍可使用」，問題點在哪？

A: 一樣是「使用者存取管理」。臨時帳號應設定自動過期或人工定期覆核移除。

Q3) 為什麼 ISO 27001 特別強調帳號生命週期管理？

A: 因為「帳號」是駭客與內鬼最主要的攻擊途徑，沒用的帳號（殭屍帳號）是最大的資安後門。
