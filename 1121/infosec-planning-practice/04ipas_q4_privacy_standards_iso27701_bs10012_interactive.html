4. 常見的資訊安全標準中，下列哪些標準可以為個人資訊保護或隱私資訊管理提供指引？（複選）

第一步：找關鍵字
  • 複選題：暗示答案可能不只一個。
  • 個人資訊保護：俗稱個資保護，指保護可識別個人的資料。
  • 隱私資訊管理：用制度來管個資的蒐集、使用與保存。
  • 提供指引：尋找「專門」指導隱私保護的標準。
第二步：白話翻譯

這題在問：「下面這四個標準中，哪些是專門教怎麼管個資、保護隱私的？（請選出所有符合的）」

第三步：解題思路

考點： 分辨「一般資安標準」與「隱私/個資專用標準」。

解題步驟：

  1. 先分辨哪些標準「專門談隱私/個資」。
  2. 再排除「只談一般資安」(如 27001) 或「談工控安全」(如 62443) 的標準。
  3. 因為是複選題，要確認「所有符合的都要選」。

原理： 隱私管理（Privacy）是資安的一個子領域，不是所有 ISO 27xxx 系列都直接處理個資。

第四步：選項分析
  • (A) ISO 27001 標準：
    這是「資訊安全管理系統」(ISMS)。重點在保護資產不外洩，雖涵蓋廣泛但非專門針對隱私指引。
  • (B) ISO 27701 標準：
    這是「隱私資訊管理系統」(PIMS)。它是 ISO 27001 的擴充，專門針對隱私保護與個資管理提供指引。
  • (C) BS 10012 標準：
    這是英國的「個人資訊管理標準」。這是全球早期針對個資保護的重要驗證標準。
  • (D) IEC 62443 標準：
    這是「工業控制系統資安」(OT Security)。主要保護工廠機器與基礎設施，與個資隱私關聯最低。
觀念補充與延伸
  • 核心定位：本題屬於「隱私與個人資料保護」章節。
  • 標準目的簡表：
    • ISO 27001：保護資料安全（不被駭）。
    • ISO 27701：保護個人隱私（合法使用資料）。
    • BS 10012：英國版個資管理標準。
    • IEC 62443：保護工廠與關鍵設施運作。
觸類旁通（舉一反三）
  • 對比概念：ISMS（資訊安全管理系統） vs PIMS（隱私資訊管理系統）。
  • 變化題型：如果考題問「與歐盟 GDPR 法規最相關的標準」，答案通常也是 ISO 27701 或 BS 10012。
記憶口訣
「隱私要管人，27701、10012」
常見陷阱
  • 看到 ISO 27001 就直覺選：它是一般資安，不是隱私專用。
  • 不認識 BS 10012：因為不是 ISO 開頭容易被忽略，但它是個資界的權威標準。
  • 誤選 IEC 62443：數字看起來很專業，但它是管機器的，不是管人的。
自我檢測
若題目改問「一般資訊安全管理制度」，會選哪一個？
ISO 27001
ISO 27701 是建立在哪一個標準之上？
ISO 27001 (PIMS 是 ISMS 的擴充模組)
為什麼 IEC 62443 不適合用來管個資？
因為它是針對工業控制系統（OT）的標準，重點在機器的運作安全，而非人的資料隱私。
