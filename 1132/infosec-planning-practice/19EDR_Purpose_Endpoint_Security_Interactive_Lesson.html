資安短期衝刺班：端點安全架構解析

19. 規劃端點安全架構時，納入端點偵測及應變機制（Endpoint Detection and Response，EDR）的最主要目的之敘述，下列何者正確？

（單選題）

第一步：找關鍵字

使用螢光筆標示並解釋以下關鍵字為何是解題線索：

  • 端點安全架構：題目限定範圍在「端點」（Endpoint），而非整個網路環境。
  • EDR：主角是「偵測與應變」系統。
  • 最主要目的：可能會有多個功能看似相關，但要選出最核心的那一個。

明確指出：本題不是在問「網路防禦」，而是在問「端點層級要做什麼事」。

第二步：白話翻譯

用高中生日常口語重新說明題目：

「公司在保護員工電腦時，為什麼要裝 EDR？它最重要是在做哪一件事？」

專有名詞白話解釋：

  • 端點：就是員工使用的電腦、筆電或伺服器。
  • EDR：即時監看電腦有沒有被入侵，並能快速處理的系統。

第三步：解題思路

這題在考什麼觀念？

一句話說明：EDR 的核心價值在「偵測與回應端點入侵行為」。

解題步驟

  1. 先確認 EDR 是部署在哪一層（端點）。
  2. 再確認它的主要功能是「看見、記錄、回應」。
  3. 排除屬於網路設備或資料管理的選項。

原理與規則（為什麼？）

  • EDR 的功能重點包括：
    • 持續監控端點行為。
    • 偵測異常或入侵跡象。
    • 提供即時回應（隔離、阻斷、分析）。
  • 因此分類如下：
    • 防禦網路入侵 → 偏向防火牆、IPS（入侵防禦系統）。
    • 管理流量或資料 → 偏向網管或 DLP（資料外洩防護）。
    • 監控端點入侵 → 正是 EDR 的核心。

生活比喻

EDR 就像電腦裡的「保全＋監視器」。
它不是在守大門（那是防火牆的工作），而是在看屋內（端點）有沒有人偷偷闖入或是有異常行為。

第四步：選項分析

  • (A) 防禦網路入侵行為：
    不對。這是防火牆或 NIDS/NIPS（網路入侵偵測/防禦系統）的工作範圍。
  • (B) 監控端點入侵行為：
    正確。EDR 的核心就是監控並回應端點上的入侵與異常行為。
  • (C) 管理端點網路流量：
    不對。這通常屬於網路流量控制設備或軟體的功能。
  • (D) 管理端點資料傳輸：
    不對。這主要是 DLP（資料外洩防護）或存取控制的功能。

觀念補充與延伸

核心觀念定位

本題屬於「端點安全 / EDR 與傳統防毒差異」基礎必考題。

重要對照

工具 主要機制 重點
傳統防毒軟體 比對已知特徵碼 阻擋已知的病毒
EDR 監控行為、異常分析 偵測未知威脅、即時回應

變化題型

  • 若問「即時回應端點威脅」→ 選 EDR
  • 若問「阻擋外部攻擊流量」→ 選 防火牆 / IPS

記憶口訣

幫助記住答案 B

「EDR 顧端點，看行為」

常見陷阱

  • 學生最容易搞混：把所有資安設備都當成「防禦入侵」，分不清楚是用來擋網路的還是擋端點的。
  • 心理盲點：忽略題目強調的是「端點」而非「網路」。
  • 避坑指南：看到 EDR，先在腦中對應兩個字：「偵測、回應」。

正確答案

答案：B

一句話理由：「EDR 的主要目的在於持續監控並回應端點上的入侵與異常行為。」

自我檢測

點擊問題查看參考答案：

小題 1：EDR 與傳統防毒最大的差別是什麼？

參考答案：是否能即時監控行為並回應（防毒主要靠特徵碼比對，EDR 靠行為分析）。

小題 2：如果要管控資料外洩，應該考慮哪一類技術？

參考答案：DLP（Data Loss Prevention，資料外洩防護）。

小題 3：如果重點在阻擋外部攻擊流量，應優先部署哪一類設備？

參考答案：防火牆或 IPS（入侵防禦系統）。
