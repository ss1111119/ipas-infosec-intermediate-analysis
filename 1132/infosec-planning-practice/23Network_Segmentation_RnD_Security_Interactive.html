題目 23

由於業務部門員工常需透過公司配發的筆記型電腦，在公司外進行工作，為避免因設備遺失導致資料外流，試問下列何種措施最為合適？

  • (A) 於筆記型電腦的 BIOS 設定並啟用開機密碼
  • (B) 於筆記型電腦設定並啟用 BitLocker 與使用者密碼登入
  • (C) 於筆記型電腦的作業系統設定並啟用 PIN 碼登入
  • (D) 於筆記型電腦的作業系統設定並啟用 Windows Hello 登入
【第一步：找關鍵字】

同學們請拿起螢光筆，把題目裡的這幾個詞畫起來：

公司外工作：代表環境不可控，沒有公司防火牆或門禁保護。

筆記型電腦：體積小，容易遺失或被偷竊。

設備遺失：這是最壞的情況，電腦已經不在我們手上了。

資料外流：我們怕的不是損失硬體（筆電），而是裡面的機密文件被看光光。

最為合適：每個選項可能都有點用，但我們要選「最有效」的那個。

老師點評：這題的核心不是「登入方不方便」，而是當設備落入壞人手中時，資料本身是否還能被讀取。

【第二步：白話翻譯】

如果用高中生聽得懂的話來說，這題其實在問：

「如果公司的筆電不小心掉在捷運上或是被偷了，哪一種設定可以確保撿到的人，就算把電腦拆開，也完全看不到裡面的資料？」

專有名詞大補帖：

BitLocker：
這是一種「全磁碟加密」技術。就像把整個硬碟鎖進一個保險箱，如果沒有鑰匙（金鑰），硬碟裡的檔案就是一堆亂碼，完全無法閱讀。
BIOS 密碼：
這是在電腦開機進入 Windows 之前的一道鎖。但它只鎖住「這台機器無法開機」，並不保護硬碟裡的資料。
PIN / Windows Hello：
這些是「登入方式」。PIN 是短密碼，Hello 是刷臉或指紋。它們是為了方便你登入，不等於資料有被加密。
【第三步：解題思路】

面對這種「裝置遺失」的題目，請依照這個邏輯流程來判斷：

  1. 思考風險：電腦被偷了，小偷可以做什麼？
    • 小偷可以把硬碟拆下來，裝到另一台電腦上讀取。
  2. 判斷防禦：
    • 如果不讓登入（PIN/Hello/BIOS）：小偷只要拆硬碟，資料就被看光了。
    • 如果資料加密（BitLocker）：小偷拆了硬碟，看到的也是亂碼。
  3. 結論：唯一能防止硬碟被拆下讀取的，只有「加密」。
【第四步：選項分析】
(A) BIOS 開機密碼
原因： 這只能擋住「開機程序」。駭客只要把筆電拆開，拿出硬碟，透過轉接線接到另一台電腦，就能直接讀取所有檔案，完全無防護力。
(B) BitLocker + 密碼
原因： BitLocker 是「整碟加密」。即使駭客拆下硬碟，因為沒有修復金鑰，硬碟內容對他來說只是無意義的亂碼。這是最有效的防護。
(C) PIN 碼登入
原因： PIN 碼只是取代長密碼的登入手段。如果沒有搭配 BitLocker，硬碟裡的檔案本身是未加密的明文，拆硬碟一樣會外洩。
(D) Windows Hello
原因： 指紋或人臉辨識（生物特徵）是為了便利性。同樣的道理，如果不加密硬碟，繞過作業系統直接讀硬碟資料是輕而易舉的。
【觀念補充與延伸】

本題屬於資安領域中的：端點安全 (Endpoint Security) 與 資料外洩防護 (DLP)。

核心原則是：

  • 移動裝置（筆電、隨身碟）視為「必然會遺失」。
  • 防護底線是「遺失後，確保資料不可讀」。
  • 關鍵技術：全磁碟加密 (Full Disk Encryption, FDE)。
口訣：「筆電會不見，資料要加密」
常見陷阱區（同學請注意！）

1. 不要把「登入驗證」當成「資料加密」：有設密碼不代表檔案被加密，拆硬碟是資安攻擊的基本招式。

2. BIOS 密碼不是萬靈丹：很多同學以為電腦打不開就安全了，卻忘了硬體是可以拆卸的。

正確答案：(B)
理由：BitLocker 提供整顆硬碟加密，即使設備遺失也能防止資料被讀取。
【自我檢測】

遮住答案，試著回答下列問題，確認你是否真的懂了：

Q1. 為什麼 BIOS 密碼不能防止資料外洩？（點擊查看答案）

答：因為 BIOS 密碼只限制電腦的啟動過程。攻擊者可以物理性地拆下筆電硬碟，將其作為外接硬碟連接到另一台電腦，繞過 BIOS 檢查直接讀取檔案。

Q2. 如果我有設 Windows 登入密碼，還需要 BitLocker 嗎？（點擊查看答案）

答：需要。Windows 登入密碼只是作業系統層級的門禁。如果硬碟沒有加密（如 BitLocker），攻擊者不需要登入 Windows，直接讀取硬碟磁區就能獲取資料。

Q3. 手機遺失通常比較不用擔心資料被拆出來讀，為什麼？（點擊查看答案）

答：因為現代智慧型手機（iOS 和 Android）預設都已經啟用了全機加密，且儲存晶片通常焊死在主機板上，不像傳統筆電硬碟容易拆卸讀取。
