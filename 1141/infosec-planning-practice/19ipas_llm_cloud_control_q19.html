iPAS 資安觀念題：雲端 LLM 服務的資料外洩風險控管

題目 19：

由於企業對於大型語言模型（LLM：能理解與生成文字/程式的 AI 模型）的使用需求大增，企業內的程式開發與研發團隊使用前述技術的雲端服務也日益頻繁，諸如 ChatGPT 及 Copilot 等等。然而為了避免公司機密資料外洩，高階管理階層要求加強對於程式開發與研發團隊使用雲端大型語言模型服務加強控制，請問下列做法何者較「不」適切？

(A) 強化微分段架構安全
(B) 強化 Web 應用層防火牆（WAF：保護網站/網頁應用的防火牆）的安全政策，尤其是程式片段的上傳
(C) 清查 LLM 的使用情形，限制非核可 LLM 服務的使用
(D) 加強雲端 API 連結使用的確認與管理

微分段 (Micro-segmentation) 主要是切分「開發者電腦」旁邊的內部網路，無法直接擋住往右邊傳送的資料。

第一步：找關鍵字
  • 雲端大型語言模型服務（ChatGPT / Copilot）
  • 公司機密資料外洩
  • 高階管理階層要求加強控制
  • 程式開發與研發團隊
  • 較「不」適切

解釋：為什麼本題關鍵在「控制雲端 LLM 使用行為與資料流向」，不是一般內網分段。

第二步：白話翻譯

情境：公司老闆很怕工程師為了貪圖方便，把公司的機密程式碼或文件直接「貼上」去問 ChatGPT 或 Copilot，結果導致機密被 AI 學走或外流。

問題：題目問下列哪一個做法對「防止資料被貼去雲端」幫助最小，或者方向根本搞錯了？

第三步：解題思路

這題在考什麼觀念？
LLM 雲端服務的資料外洩風險控管與控制點選擇。

解題步驟：

  1. 先鎖定「資料如何出去」：透過瀏覽器上傳內容、貼上文字、或是程式編輯器裡的 API 自動連結。
  2. 找能直接限制/監控 LLM 使用的控制措施：包含清查有哪些人在用、列出核可清單、管理 API 金鑰、監控上傳內容。
  3. 排除偏向「內部東西彼此隔離」的做法：找出只能管內網，管不到「對外雲端服務」的選項。

原理與規則（白話）：

  • 資料外洩的核心是「誰把什麼資料送到哪裡」。
  • 控制要落在「雲端服務使用管控、資料上傳/分享、API 管理與稽核」。
第四步：選項分析
選項 控制點 適切性 白話原因
(A) 強化微分段架構安全 內部網路 不適切 微分段（Micro-segmentation：把內部網路切很細）主要在防範駭客進來後到處亂跑（橫向移動）。但工程師主動把資料貼去雲端，是「對外」的行為，切分內網擋不住這件事。
(B) 強化 WAF 與程式片段上傳政策 邊界/Web流量 適切 WAF 可以設定規則，檢查出去的 Web 流量有沒有包含像程式碼特徵的敏感資料，降低被不當上傳的風險。
(C) 清查與限制非核可 LLM 管理/治理 適切 這是「治理」（Governance）。先規定只能用哪幾家安全的 AI，禁止亂用不知名的服務（Shadow IT），直接減少風險源頭。
(D) 加強雲端 API 連結管理 程式介面 適切 現在寫程式很多是用 Copilot 插件透過 API 連出去。管好 API 權限跟連線對象，才能知道資料有沒有被偷偷傳走。
第五步：觀念補充與延伸

核心觀念定位

  • 新興科技風險管理 → 生成式 AI / LLM 使用治理
  • 考試常見度：中高（近年趨勢題增多）

定義整理（白話）

  • LLM：大型語言模型，能生成文字與程式碼的模型（如 ChatGPT）。
  • Shadow IT（影子 IT）：員工私自使用公司未核准的雲端工具或軟體。
  • 微分段（Micro-segmentation）：把內網切得很細，像潛水艇的隔艙，一艙進水（中毒）不會淹沒全船，主要防內部擴散。

觸類旁通

  • 網路分段（管內部互通） vs 雲端服務治理（管對外使用與資料流）。
  • WAF（保護 Web 流量） vs CASB（雲端存取安全代理：專門管雲端服務使用的工具，這也是常見考點）。
第六步：記憶口訣

「怕貼上雲端 AI：
先管用哪家、再管傳什麼、最後查 API；
內網切太細不一定對題。」

第七步：常見陷阱
  • 易錯點：看到選項 (A) 有「安全」兩個字，而且「微分段」聽起來技術很深奧，就不敢選它（以為它是對的）。
  • 心理盲點：以為任何「架構強化」都能解資料外洩。其實資安控制措施必須「對症下藥」。
  • 避坑指南：
    • 先畫資料流：資料要去哪？怎麼出去？誰在用？
    • 只要問題是「外送到雲端」，優先選「治理 / 上傳管控 / API 管理」。

正確答案：(A)

一句話理由：微分段偏向內部網路隔離，對「使用雲端 LLM 服務造成的資料外送」不是最直接有效的控制點。

自我檢測

1) 如果公司允許使用 ChatGPT，但禁止貼上機密程式碼，除了政策宣導外還能做哪些技術控管？

參考答案：
建立核可工具清單與代理伺服器、上傳內容檢測/阻擋（例如 DLP 資料外洩防護或 Proxy 規則）、記錄稽核、敏感資料標記與權限控管等。

2) 為什麼「限制非核可 LLM」能降低風險？

參考答案：
可避免員工把資料送到未知或不受控的平台（那些平台可能會拿你的資料去訓練），降低資料外洩與合規風險，並集中管理與稽核。

3) Copilot 類工具的 API 管理，主要在管什麼？

參考答案：
管 API 權限（誰可以用）、可存取的資料範圍（能讀哪些 Code）、金鑰/Token 使用狀況、連線對象與稽核紀錄，避免過度授權與資料被背景程式拉走。
