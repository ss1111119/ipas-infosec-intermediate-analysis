標準風險管理

下列何項是美國國家標準技術研究院（NIST）的網路安全框架（CyberSecurity Framework，CSF）從版本 1.1 更新到 2.0 中最明顯的改變？
查看詳細解析
第一步：找關鍵字
  • NIST CSF：美國官方資安標準框架，考的是「標準」非產品。
  • 版本 1.1 → 2.0：明確在考「版本差異」。
  • 最明顯的改變：不是細節調整，而是「一眼就看得出來的結構變化」。
第二步：白話翻譯

這題在問：「NIST 的 CSF 從舊版 1.1 升級到新版 2.0，到底『最大、最關鍵』的差異是什麼？」

不是問有沒有修錯字或多寫說明，而是問整個框架層級有沒有不一樣。

第三步：解題思路與生活例子

1. 原理與規則：

  • CSF 1.1 (舊版)：只有五大功能 (Identify, Protect, Detect, Respond, Recover)，重點在「事情怎麼做」。
  • CSF 2.0 (新版)：新增第六功能 Govern (治理)。這代表 NIST 告訴你，資安不只是工程師的事，更是「管理階層要負責的事」。

2. 生活化比喻 (學校與校長)：

  • 五大功能 (舊)：就像學校老師負責教學、點名、補救學生 (第一線執行)。
  • Govern 治理 (新)：就像校長室與董事會。要不要編預算？要不要請資安人員？出事誰負責？CSF 2.0 就是多加了一層「校長室」。
第四步：選項分析
  • (A) 錯誤：這五大功能本來就在 1.1 版裡，屬於「舊內容」，非新增。
  • (B) 正確：Govern（治理）是 CSF 2.0 才新增的第六大核心功能，完全符合「最明顯的改變」。
  • (C) 錯誤：定義確實有調整，但那是細節優化，不符合「最明顯」關鍵字。
  • (D) 錯誤：CSF 是「高層框架」，不是雲端技術操作手冊。
觀念補充與延伸

法條或定義 (白話版)：
Govern (治理) = 確保組織在資安上有明確責任、決策流程與高層支持。白話說：不是出事才找人扛，而是一開始就講清楚誰負責。

觸類旁通 (舉一反三)：

  • 對比概念：CSF (方向與治理框架) vs ISO 27001 (制度與稽核導向)。
  • 常見變化題：若問「哪一項最能體現高層資安責任？」，答案也是選與「治理 (Govern)」相關的選項。
記憶口訣與常見陷阱

記憶口訣：
「CSF 2.0，多一個管事的」 或 「五加一，治理進來了」。

常見陷阱 (避坑指南)：

  • 心理盲點：太熟舊版的五大功能，看到選項 (A) 列出五大功能就覺得好親切直接選，結果掉入陷阱。
  • 避坑指南：看到「CSF 2.0」關鍵字 → 請立刻先找 Govern (治理)。
自我檢測
Q1: 為什麼 NIST 要在 CSF 2.0 特別加入「治理」？
參考答案：
因為資安需要高層決策與責任，不只是技術問題。若缺乏治理，資安將缺乏資源與方向。
Q2: 如果一個組織只有技術防護，沒有高層支持，CSF 2.0 認為會出什麼問題？
參考答案：
會導致資源不足、責任不清、風險無法有效管理。
Q3: 「Govern」最接近下列哪一個角色？(A)工程師 (B)事件處理人員 (C)高階主管
正確答案：(C) 高階主管
治理層級涉及決策與資源分配，屬於管理階層的職責。
