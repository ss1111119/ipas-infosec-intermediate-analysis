情境背景

勒索軟體與資料外洩攻擊之準備、預防、緩解與應變之實務上，可透過分析系統執行特定指令來進行威脅獵補（Threat Hunting，就是主動在系統中找尋「可能尚未被發現的攻擊行為」），以即早偵測並發現異常行為。

Q9. 下列何種 Windows 指令「非」勒索軟體常用於竄改系統檔案／備份之目的？

查看詳細解析
Step 1找關鍵字
  • 勒索軟體（Ransomware）：加密資料勒索贖金的惡意程式。
  • 資料外洩（Data Exfiltration）：把資料偷偷傳出去。
  • 威脅獵補（Threat Hunting）：主動找「可疑行為指令」，不是等警報。
  • 竄改系統檔案 / 備份：勒索軟體最愛先「破壞備份」，讓你沒得還原。
  • Windows 指令：考的是「哪些是勒索軟體常濫用的內建工具（LOLBins）」。
Step 2白話翻譯

勒索軟體在動手加密之前，通常會先把 Windows 裡的「備份、還原點」刪掉或破壞。題目在問：下面這些 Windows 指令中，哪一個「不是」勒索軟體常拿來做這種壞事的？

Step 3解題思路

原理與規則：勒索軟體最怕你做什麼？→「從備份還原」。所以它們會優先執行刪除 Volume Shadow Copy（磁碟陰影複本）或 Windows Backup 的指令。

生活例子

就像綁匪綁架前，會先把你的備用鑰匙扔掉（刪除備份），讓你進不去家門（無法還原），只能乖乖交出贖金。

解題步驟：找出選項中那些指令是「刪除/破壞備份」的常客，剩下那個「幾乎沒聽過、也不常見於攻擊行為的」，通常就是答案。

Step 4選項分析
  • (A) fsutil.exe 可疑
    檔案系統管理工具。攻擊者常用它刪除 USN Journal，藉此隱藏檔案修改痕跡。
  • (B) ntfsbackup.exe 正確答案
    這個指令並非 Windows 常見或實務上被勒索軟體濫用的標準工具，也不是威脅獵補中常監控的指令。
  • (C) vssadmin.exe 極度危險
    勒索軟體經典指令。常見行為：vssadmin delete shadows /all /quiet（安靜刪除所有備份陰影）。
  • (D) wbadmin.exe 高風險
    Windows 備份管理工具。常用於刪除 Catalog（目錄）或停止備份服務。
觀念補充與延伸

LOLBins (Living off the Land Binaries)：

指攻擊者「就地取材」，直接使用受害電腦裡本來就有的合法工具（如 PowerShell, Certutil, Vssadmin）來幹壞事。這樣可以降低被防毒軟體偵測到的機率，因為這些工具本身是合法的。

觸類旁通（舉一反三）

除了題目提到的，還有哪些指令要盯緊？

  • bcdedit：修改開機設定，關閉自動修復。
  • wevtutil：清除 Windows 事件紀錄（Event Logs），以此湮滅證據。
  • net stop：停止防毒軟體或備份服務。
記憶口訣
「勒索先刪備份，VSS 跟 WB 要盯緊；
沒聽過的那一個，考試多半就是你。」
常見陷阱

心理盲點：看到選項裡有 "backup" 字樣（如 ntfsbackup），就直覺以為它一定跟備份有關，肯定會被勒索軟體攻擊。

避坑指南：題目問的是「非」常用指令。你要選的是那個在資安報告、威脅獵補清單中「最生面孔」的選項。

自我檢測與反思

Q1: 為什麼勒索軟體攻擊前，常會先刪除 Volume Shadow Copy？

點擊查看答案
因為只要磁碟陰影複本（還原點）還在，使用者就有機會不付贖金直接還原資料。刪除它，勒索才能構成最大威脅。

Q2: 若在端點監控中看到 vssadmin delete shadows 指令，資安人員應立即做什麼？

點擊查看答案
這屬於高風險指標（IOC）。應立即通報、隔離該主機網路（斷網），保全現有記錄，並啟動事件應變（IR）流程。

Q3: 為什麼威脅獵補會特別關注「系統內建工具」而不是只找陌生惡意程式？

點擊查看答案
因為內建工具（LOLBins）本身是合法的，較不易被傳統防毒軟體直接攔截，且常被攻擊者用於隱蔽攻擊行為。
