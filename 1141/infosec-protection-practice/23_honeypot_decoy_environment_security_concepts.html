單選題
關於誘餌環境（Decoys Environment，或稱蜜罐 Honeypot）的相關敘述，下列何者「最不正確」？

查看詳細解析

第一步：找關鍵字

  • 誘餌環境 / 蜜罐 (Honeypot)：本題主角，考「資安欺敵與偵測技術」。
  • 整合式 / 連接式 / 獨立式：考「不同部署方式的風險與能力差異」。
  • 最不正確：尋找一個說法「太過理想化」或「過度保證」的選項。

第二步：白話翻譯

蜜罐是用來「故意騙駭客」的假系統，不同架構風險不同。請找出哪一個說法把蜜罐講得太神、太安全、太萬能，因此是錯的。

第三步：解題思路

解題步驟

先確認每個選項有沒有「過度保證安全或效果」。看到「可完整偵測」、「可識別零日漏洞」要特別警覺。

原理與規則

資安考試原則：沒有任何機制是「完整」或「保證」的。

生活例子

蜜罐像是「釣魚用的假餌」。假餌能吸引魚，但不代表所有魚都會來吃，更不代表用了假餌就能抓光池塘裡所有的魚。

第四步：選項分析

  • (A) 正確： 誘餌本來就是假的（仿真）數位物件，如假文件、假帳號，目的是欺騙。
  • (B) 正確： 「整合式蜜網」放在生產環境內，雖然真實度高，但若被攻破，攻擊者確實可能竊聽生產環境的流量，風險較高。
  • (C) 錯誤（本題答案）：
    錯在「可完整偵測」與「識別零日漏洞」。蜜罐只能偵測「針對該蜜罐」的行為，無法保證「完整」偵測所有新攻擊。
  • (D) 正確： 「獨立式蜜網」與生產環境隔離，就算被攻破也只壞在沙箱裡，不影響正式系統，因此風險相對最低。

觀念補充與延伸

法條或定義（白話）

Honeypot（蜜罐）：單一的誘捕系統。
Honeynet（蜜網）：多個蜜罐組成的誘捕網路環境。

觸類旁通（舉一反三）

對比概念：蜜罐偏向「偵測與分析」；防火牆 / WAF 偏向「阻擋與防禦」。
變化題型：若問「哪一種蜜網風險最低」？選獨立式。若問「哪一種最貼近真實攻擊」？選整合式。

常見陷阱

心理盲點

容易被「零日漏洞」四個字嚇到，以為功能寫得很高級就是對的。

避坑指南

考試看到這些絕對詞彙要小心：完整、完全、保證、一定，在資安領域中通常是錯誤選項。

記憶口訣

「蜜罐能釣魚，但不能保證全釣到。」看到「完整偵測」、「完全防護」，九成是錯的。

自我檢測

Q1：為什麼獨立式蜜網 (Standalone) 風險最低？

顯示答案
因為它不連接正式生產系統，攻擊影響被隔離在獨立環境中。

Q2：蜜罐能不能取代防火牆？

顯示答案
不能。蜜罐是「誘捕與偵測」，防火牆是「阻擋與防禦」，角色不同。

Q3：如果題目問「最貼近真實攻擊行為」的蜜網會選哪種？

顯示答案
通常是「整合式」或「連接式」，因為它們與真實環境有互動，較能誘騙高階駭客。
